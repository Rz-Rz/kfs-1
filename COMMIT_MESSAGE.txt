feat(i386): align build + sources to mandatory i386 target

The project requirements mandate i386 (32-bit x86). This change makes i386 the
explicit default across the build, run, and source layout.

Build/toolchain:
- Default target is now `arch ?= i386`.
- NASM outputs 32-bit ELF objects (`-felf32`).
- Link uses i386 emulation (`ld -m elf_i386`).
- Run target uses `qemu-system-i386`.

Source layout:
- Move architecture sources under `src/arch/i386/`.
- Remove the obsolete `src/arch/x86_64/` tree.

Docs/cleanup:
- Update repo status/features docs with i386 alignment proof and remaining gaps.
- Remove outdated x86_64 build artifacts and the old snapshot digest.

Proof (local):
- `make` produces `build/kernel-i386.bin`.
- `file build/kernel-i386.bin` -> `ELF 32-bit ... Intel 80386`
- `readelf -h build/kernel-i386.bin` -> `Class: ELF32`, `Machine: Intel 80386`
